# cs2_hello

A minimal CS2 (Counter-Strike 2) DLL that prints messages to the in-game console via injection. Useful as a starting point for learning game hacking or modding.

## Features

- Injects into CS2 and prints colored messages to the in-game console
- Uses `ConColorMsg` / `Msg` exports from `tier0.dll` (no vtable index guessing)
- Resolves `ICvar` pointer from [cs2-dumper](https://github.com/a2x/cs2-dumper) offsets
- Debug console window for step-by-step output

## Requirements

- Visual Studio 2022 (or compatible) with C++ desktop development
- Windows x64
- Counter-Strike 2 installed and running

## Build

1. Open `cs2_hello.slnx` in Visual Studio
2. Select **x64** configuration
3. Build → Build Solution (Ctrl+Shift+B)

Output: `cs2_hello\x64\Debug\cs2_hello.dll`

## Usage

1. Start CS2 and load into a map (or main menu)
2. Inject `cs2_hello.dll` into `cs2.exe` using an injector (e.g. Xenos, Extreme Injector, Process Hacker)
3. A debug console window appears with status messages
4. Press **~** (tilde) in CS2 to open the in-game console — you should see the injected messages
5. Press Enter in the debug console to detach

## Project Structure

| File | Description |
|------|-------------|
| `dllmain.cpp` | Entry point, console setup, ConColorMsg/Msg calls |
| `interfaces.hpp` | Interface offsets from cs2-dumper (e.g. `VEngineCvar007`) |
| `offsets.hpp` | Game offsets from cs2-dumper |
| `pch.h` / `pch.cpp` | Precompiled headers |
| `framework.h` | Windows headers |

## Offsets

Offsets are generated by [a2x/cs2-dumper](https://github.com/a2x/cs2-dumper). They change with game updates. Regenerate `interfaces.hpp` and `offsets.hpp` if CS2 has been patched.
